# Introduction

Kevara is a sovereign professional trust portfolio built natively on AT Protocol. These docs cover the technical infrastructure: the Trust Gateway API, the AT Protocol lexicons Kevara reads and writes, and the open standards that underpin portable professional identity.

<Note>
  Kevara is in active development. These docs reflect current implementation alongside the near-term roadmap. Sections marked with a note callout are specified but not yet live.
</Note>

## Who these docs are for

**Developers** building on AT Protocol who want to read Kevara profile data, subscribe to the Kevara Labeler, or integrate Trust Gateway verification into their own tools.

**Institutions** evaluating the Trust Gateway as an Open Badges 3.0 or W3C VC issuance and verification channel.

**Professionals** who want to understand exactly what data Kevara reads and writes on their behalf, and how the sovereignty guarantee works technically.

## How the pieces connect

| Layer | What it does | Where to read |
|---|---|---|
| AT Protocol | Identity portability and DID-based ownership | [atproto.com](https://atproto.com) |
| SIFA Standard | Open schema for portable professional data | [Lexicon Reference](/lexicon-reference) |

| Ozone Labeler | Cryptographically signed trust labels | [Trust Gateway](/trust-gateway) |
| Trust Gateway | Stateless verification layer for institutions and enterprise | [Trust Gateway](/trust-gateway) |

## Core principle

Kevara is a stateless display layer. It reads from and writes to your Personal Data Server. It holds no copies of your data. Your DID, your records, your credentials — they are yours.
